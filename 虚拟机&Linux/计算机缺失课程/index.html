shell

计算机为用户提供了多种交互接口，最常见的是图形界面，而命令行界面（CLI）则提供了另一种交互方式。屏幕上打开的 Terminal（终端模拟器，如 Windows Terminal、iTerm2）只是一个显示文字的窗口，真正在里面听候差遣、负责解析命令的，是 Shell。

  1. 操作系统中的“夹心饼干”架构

在操作系统架构中，Shell 处于“用户”与“Kernel（内核）”之间：

  • 内核：操作系统的核心，不直接面向普通用户交互，它专注于底层硬件管理、内存分配和进程调度。
  • Shell（壳层）：包裹在内核外面的一层。广义上，所有为用户提供操作界面的程序都叫 Shell（比如 Windows 的资源管理器/桌面本质上就是图形 Shell）。但狭义上，我们通常指命令行界面的解析器，它负责接收用户输入的文本指令，将其翻译成系统调用，交给内核执行，最后把结果返回给用户。
  1. 外观极简，定制极高

虽然命令行看起来“定制化低”（都是黑底白字，没有按钮和图标），但在功能和工作流上，Shell 是定制化极高的。通过编写配置文件（如 .bashrc、.zshrc），用户可以自定义快捷键、环境变量、命令别名，甚至通过诸如 Oh My Zsh 这样的框架，把命令行打扮得非常炫酷且高效。

  1. 作为编程语言的 Shell

Shell 本质上也是一种解释型编程语言（最常见的是 Bash）。它不像 C 或 Java 那样用来编写大型应用软件，而是用来编写“胶水语言”。它内置了变量、条件判断、循环等逻辑控制能力，主要用于：

  • 自动化运维：批量重命名文件、定时备份数据。
  • 流程编排：将各种零散的命令行工具（如 grep、awk、sed）像管道一样拼凑起来，处理复杂的数据流。
  1. Shell 的家族生态

Shell 并不是特指某一个软件，而是一个庞大的家族。不同的操作系统或用户偏好催生了不同的 Shell 实现：

  • **sh (Bourne Shell)**：最古老的 Unix Shell，是所有现代 Shell 的祖宗。
  • **bash (Bourne Again Shell)**：大多数 Linux 发行版默认的 Shell，生态最完善，绝大多数脚本都兼容它。
  • zsh：在 macOS (Catalina及以后) 中成为默认 Shell，功能比 bash 更强大，插件生态极其丰富（配合 Oh My Zsh）。
  • PowerShell：微软开发的现代 Shell，不仅处理文本，还能处理对象，深度集成 Windows 生态（.NET）。
