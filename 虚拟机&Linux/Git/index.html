常用操作

设置用户签名： git config –global user.name 用户名

设置用户邮箱： git config –global user.email 邮箱

查看本地库状态： git status  (是否有修改、现在的分支等)

从云端获取文档： git clone  +<URL>

添加改动到暂存区: git add .

提交修改到本地仓库: git commit

上传修改到云端: git push

有不同，但是直接硬推：git push origin master -f

查看未提交的不同： git diff

查看提交记录： git log  (如果看提交编号的话用：git log –pretty=oneline）

回退版本： git reset  (git reset –hard 版本号)（可以写HEAD，head表示当前版本，head^表示上一个，head^^上上个）

查看上一次提交：git show（或者git log -n1 -p）

回退之后再找回：git reset –hard 要退回的那个版本号

每一次命令记录（提交版本记录）：git reflog

把某个文件从暂存区里删掉： git rm –cached 文件名

查看工作区和版本库里最新版本的不同： git diff HEAD –文件名

撤销工作区中的修改： git checkout – 文件名 （包括复原删除的修改）

查看当前分支：git branch

查看分支：git branch -v

切换分支：git checkout 分支名

删除分支：git branch -d 分支名

合并指定分支到当前分支（正常合并）：git merge 分支名

合并指定分支到当前分支（冲突合并）：git

碰到冲突的时候，直接merge会生成一个新的临时分支，在这个分支中用git status查看有冲突的文件，并打开有冲突的文件，可以看到git系统标出的那些字段有不同。可以直接在里面进行修改。再把这个临时文件add、commit（不用写文件名，直接addcommit）

切换到新（新建）分支：git switch -c 分支

切换到已有的master分支：git switch master

配置别名（重命名指令）：配置别名 - 廖雪峰的官方网站 (liaoxuefeng.com)

搭建Git服务器 - 廖雪峰的官方网站 (liaoxuefeng.com)

tip：提交信息写错了

如果你的提交信息(commit message)写错了且这次提交(commit)还没有推(push), 你可以通过下面的方法来修改提交信息(commit message):

Git commit –amend –only

或者一条搞完：

git commit –amend –only -m ‘xxxxx’

特殊：git rebase  也是合并分支，但是和merge不同

三、远程仓库操作

查看当前所有远程地址别名： git remote -v

起别名： git remote 别名 远程地址

推送本地分支到远程仓库： git push 别名 分支

拉取远程库到本地：git pull 别名 分支   （这个动作会自动提交本地库）
