浏览器的进程模型

何为进程？

程序运行需要自己专属的内存空间，这块空间可以简单理解为进程。进程之间相互独立隔离，即使需要通信，也需要双方同意。
![[Pasted image 20240922151426.png]]

线程

有了进程就可以运行程序代码，运行代码的“人”称为“线程”。一个进程至少一个，一个进程可以包含多个线程。

浏览器的进程和线程

浏览器是一个多进程多线程的应用程序，内部极其复杂，为了减少连环崩溃的几率，启动浏览器之后，它会自动启动多个进程。在浏览器的任务管理器能看到。
![[Pasted image 20240922151842.png]]
![[Pasted image 20240922151937.png]]
1、浏览器进程

负责界面显示、用户交互、子进程管理等，浏览器进程内部会启动多个线程处理不同的任务。

2、网络进程

负责加载网络资源。网络进程内部会启动多个线程处理不同的网络任务。

3、渲染进程

渲染进程启动后，会开启一个渲染主线程，主线程负责执行HTML、CSS、JS代码。默认情况下浏览器回味每个标签开启一个新的渲染进程，以保证不同标签页指间不会相互影响。

JS事件

发表于 2024-09-10 本文字数： 1.5k 阅读时长 ≈ 1 分钟

JS使用的是事件驱动的设计模式，事件（event）是JS和HTML交互的基础，本质是程序各个组成部分之间的一种通信方式，也是异步编程的一种实现。任何文档或者浏览器窗口发生的交互都要通过绑定事件进行。可以详细分解为触发谁的事件、触发什么事件、触发事件之后做什么。按照这样的思路去写事件处理。常见的事件包括onclick/onload/onscroll/onresize等。

DOM节点的事件操作（监听和触发）都定义在EventTarget接口，所有节点对象都有这个接口，一些浏览器内置对象也有（比如，XMLHttpRequest、AudioNode、AudioContext）。
该接口主要提供三个实例方法。

  • addEventListener()：绑定事件的监听函数
  • removeEventListener()：移除事件的监听函数
  • dispatchEvent()：触发事件

这里的dispatch就在当前节点上触发指定事件，从而出发监听函数的执行。返回值是布尔值，有调用就false，表示未被取消。

TIP：DOM级别

DOM标准的三个版本，DOM123，DOM1是HTML和XML文档底层结构，DOM2和3在此基础上引入了更多交互能力，支持更高级的XML特性。

事件模型：

事件模型就是通过监听函数（listener）对事件做出反应。事件发生之后浏览器监听到，就会执行相应函数。JS中有三种方法绑定监听函数，HTML的on属性、元素节点的事件属性（也是on开头的，比如window.onload）、EventTarget.addEventListener()。

this:

监听函数内部的this指向触发事件的那个元素节点。

事件的传播机制：

一个事件发生之后，会在子元素和父元素之间传播（propagation）。这种传播分成三个阶段。

  • 第一阶段：从window对象传导到目标节点（上层传到底层），称为“捕获阶段”（capture phase）。
  • 第二阶段：在目标节点上触发，称为“目标阶段”（target phase）。
  • 第三阶段：从目标节点传导回window对象（从底层传回上层），称为“冒泡阶段”（bubbling phase）。

这种三阶段的传播模型，使得同一个事件会在多个节点上触发。

也叫事件委托，就是把要做的事情委托给别人来做，由于冒泡机制，点击子元素的同时会触发父元素的事件，那也就可以把子元素的任务交给父元素做，由父节点的监听函数统一处理多个子元素的事件。

要实现夜间模式（又称暗黑模式），主要是通过切换页面的样式，如背景色、文本颜色等，使页面在夜间环境下更加适宜阅读。可以通过 JavaScript 动态切换 class 或使用 CSS 变量来实现这个功能。以下是几种常见的实现方法：

1. 基于 CSS Class 的切换

步骤：

  1. 定义夜间模式和日间模式的样式。
  2. 使用 JavaScript 监听切换按钮的点击事件，动态切换页面的 class，从而应用不同的样式。

实现方案：

代码解释：

  1. CSS 样式：

    • 定义了 .dark-mode 类，它会影响页面背景颜色、文本颜色等，应用夜间模式时，body 的背景颜色变为深色，文本颜色变为浅色。

  2. toggleDarkMode 函数：

    • 通过 document.documentElement.classList.toggle("dark-mode") 切换 dark-mode 类，切换夜间模式和日间模式。
    • 使用 localStorage 保存用户选择的模式，下次页面加载时自动应用。

  3. 页面加载时检查用户偏好：

    • 在页面加载时，通过 localStorage 检查用户之前是否启用了夜间模式，并相应地设置页面样式。

2. 使用 CSS 变量实现夜间模式

你可以通过 CSS 变量来更灵活地管理不同的颜色和样式。通过 JavaScript 切换根元素的 CSS 变量，可以更轻松地控制多个样式。

优点：

  • 全局控制样式：只需要改变 CSS 变量，不需要为每个元素单独定义样式。

3. 系统主题自动检测

可以通过 CSS 媒体查询检测用户的系统主题偏好，自动启用夜间模式。

这种方式让页面自动适应用户操作系统的夜间模式设置，但它无法手动切换，只能根据系统设置调整。

方法对比

方法 优点 缺点
切换 CSS 类 实现简单，适合小型项目 大量样式需要手动定义和切换
CSS 变量 灵活且可扩展，适合大型项目 浏览器兼容性要求较高
系统主题检测 自动适应系统主题 用户无法手动切换

总结

  • 使用切换 CSS 类 是一种常见的、简单的方式，适合中小型项目。
  • 使用 CSS 变量 更灵活，适合大型项目，尤其当样式复杂时，更方便维护。
  • 系统主题检测 可以提供自动适应的体验，但需要配合其他方式让用户手动切换。
