Nginx

Nginx

介绍

Nginx（发音为“engine-x”）是一个高性能的开源 Web 服务器和反向代理服务器，可以处理大量的并发连接和请求。它使用事件驱动的异步架构和多线程设计，可以高效地处理并发请求，同时也支持反向代理、负载均衡、动态HTTP缓存、SSL/TLS终止、基于模块的扩展等功能，可以用作邮件代理服务器 和 通用 TCP/UDP 负载均衡器。

Nginx 的应用场景非常广泛，以下是其中的几个：

  • Web 服务器：Nginx 可以作为 HTTP 服务器，处理并发的静态请求和动态请求，有高效的文件缓存机制，并且可以支持负载均衡和缓存，为网站提供高性能和高可用性。
  • 反向代理服务器：Nginx 可以作为反向代理服务器，接收客户端请求并将其转发到后端服务器，同时也可以支持负载均衡和缓存。
  • 邮件代理服务器：Nginx 可以作为邮件代理服务器，支持 POP3、IMAP 和 SMTP 协议，并且可以支持 SSL/TLS 加密和反垃圾邮件功能。
  • 流媒体服务器：Nginx 可以作为流媒体服务器，支持 RTMP、HLS 和 DASH 协议，可以用于实现直播、点播和视频-on-demand（VoD）等场景。
  • 安全功能：能够配置HTTPS（SSL/TLS），限制请求速率防止DDoS，也可以提供IP过滤、访问控制列表。

总之，Nginx 具有高性能、可扩展性和可靠性等特点，被广泛应用于大型网站、互联网公司、云计算、视频流媒体、游戏等领域。

工作原理

1. 事件驱动模型

  • NGINX 采用异步非阻塞的事件驱动架构，每个工作进程使用单线程。
  • 通过 epoll（Linux）或 kqueue（FreeBSD）等系统调用，高效处理大量并发请求。

2. 多进程架构

  • 包括 主进程 和多个 工作进程：
    • 主进程：
      • 负责读取和解析配置文件。
      • 启动、停止和管理工作进程。
    • 工作进程：
      • 处理实际的网络请求。
      • 独立运行，互不干扰。

优点

  1. 高性能：专为高并发场景设计，支持数万并发连接。
  2. 资源占用低：相比 Apache 等传统 Web 服务器，内存和 CPU 消耗更少。
  3. 灵活扩展：通过模块化设计，支持插件和扩展（如 Lua）。
  4. 易于配置：简洁的配置文件，易于管理。

常见问题

  1. 502 Bad Gateway：

    • 后端服务不可用或连接超时。
    • 检查 proxy_pass 配置和后端服务状态。

  2. 413 Request Entity Too Large：

    • 上传文件超过限制。
    • 解决：增加 client_max_body_size。

  3. CORS 问题：

    • 前端跨域访问被阻止。
    • 解决：在反向代理中添加 CORS 头。

  4. 高并发时性能下降：

    • 检查 worker_connections 和 worker_processes 配置，确保资源分配合理。
