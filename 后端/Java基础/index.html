堆是用于存放对象实例和数组的内存区域，是垃圾回收的主要区域。所有对象都在堆中分配内存。堆是线程共享的，所有线程都能访问堆里面的对象。堆内存通常被划分为年轻代（Young Generation）和老年代（Old Generation）。年轻代又包括 Eden 区 和两个 Survivor 区。垃圾回收器会基于不同代的特性进行回收。

  • **年轻代 (Young Generation)**：存放新创建的对象。包含 Eden 区 和两个 Survivor 区。新创建的对象首先被分配在 Eden 区，经过多次垃圾回收后，存活的对象会被晋升到老年代。
  • **老年代 (Old Generation)**：存放生命周期较长的对象。如果对象在年轻代经历了多次垃圾回收仍然存活，就会被移动到老年代。
  • 持久代 (Permanent Generation)（JDK 1.7 及之前）：存放类元数据、静态变量等。方法区通常就是指持久代。
  • **元空间 (Metaspace)**（JDK 1.8 及之后）：取代了持久代，用于存储类的元数据。

栈是用于存放局部变量和方法调用的内存区域，每个线程有自己的栈，栈内存大小有限。栈的作用是存储方法的调用信息和局部变量。

  • 线程私有：每个线程都有自己的栈，线程之间的栈是互不干扰的。每当一个方法被调用时，JVM 会在当前线程的栈中为该方法创建一个栈帧。

  • 存放局部变量：方法中的局部变量、方法参数以及方法调用时的中间数据都会被存放在栈中。局部变量的生命周期仅限于方法调用期间。

  • **栈帧 (Stack Frame)**：每次方法调用时，JVM 会为该方法分配一个栈帧，用于存储局部变量、操作数栈、返回地址等信息。栈帧会在方法执行完后被销毁。

  • 自动分配和销毁：栈内存的分配和销毁由 JVM 自动管理，栈中的数据是后进先出（LIFO）形式的。

  • 方法调用：每当方法被调用时，会在栈中分配一个新的栈帧。栈帧中包含方法的局部变量、返回地址等信息。当方法执行完成时，栈帧被销毁，栈中的空间被释放。

  • 局部变量的生命周期：局部变量和方法参数只存在于栈帧内，栈帧销毁时，它们所占用的内存也随之释放。

方法区存储了类的元数据、方法信息、常量池、静态变量等。方法区也叫做 **类区 (Class Area)**，它是所有线程共享的内存区域。方法区的内容是固定的，它存储了 JVM 执行过程中所需的类信息。

  • 存储类的元数据：方法区用于存储类的结构，如类名、父类名、访问修饰符、字段信息、方法信息、构造方法、常量池等。

  • 常量池：常量池是方法区的一部分，用于存储类中使用的常量，如字符串、数字常量等。

  • 静态变量：静态变量（static 声明的变量）也是存储在方法区中的，而非实例化对象时的堆内存。

  • 线程共享：方法区是所有线程共享的内存区域，因此它存储的信息对所有线程都是可见的。

  • 类的加载与卸载：类的加载过程会将类信息存储到方法区，类的卸载会将其从方法区中移除。

  • **常量池 (Constant Pool)**：存放类中的常量，包括字符串字面量和符号引用。常量池属于方法区的一部分。

  • 类的元数据：每个类的信息（如字段、方法、构造函数等）都被加载到方法区中。

  • 静态变量：类的静态变量也是存储在方法区中的，而不是存放在堆内存中。

方法区的内存区域是有限的。在 JDK 1.7 及之前，方法区的内存由 永久代 (Permanent Generation) 管理；从 JDK 1.8 开始，方法区被替换为 **元空间 (Metaspace)**，元空间的大小不再受限于堆大小，而是由操作系统的内存决定。

堆、栈、方法区的关系：

  1. 堆和栈：
    • 堆：存储对象实例，垃圾回收管理；是线程共享的。
    • 栈：存储方法调用的局部变量，每个线程有自己的栈；是线程私有的。
  2. 方法区：存储类的信息、静态变量、常量池等，是线程共享的。
  3. 内存管理：堆和栈的内存管理由 JVM 自动管理，栈遵循 LIFO（后进先出）原则。堆中的对象则由垃圾回收器管理。而方法区的内存包含了类信息、常量池和静态变量，负责存储程序的元数据。

总结：

  • 堆：存放对象和数组，垃圾回收管理。
  • 栈：存放方法的局部变量和调用栈信息，线程私有。
  • 方法区：存放类的元数据、静态变量和常量池，线程共享。
