理论词汇

理论词汇

  • Normative：规范性，与标准不同，通常是解释性的。
  • Semantic：语义化的

编程语言通用词汇

  • Syntax：语法，指编程语言的结构规则，例如 if...else 语句在不同语言中有特定的语法格式。
  • Statement：语句，是编程语言中完成一个基本操作的指令，如赋值语句 x = 10。
  • Expression：表达式，由变量、常量和运算符组成的式子，会产生一个值，像 2 + 3 就是一个表达式。
  • Variable：变量，用于存储数据的容器，可在程序运行过程中改变其值。
  • Constant：常量，值在程序运行过程中不会改变的量，例如在 Python 中习惯用全大写命名常量 PI = 3.14。
  • Function：函数，将一组代码封装起来，实现特定功能，可重复调用。
  • Method：方法，类中定义的函数，与特定对象相关联。
  • Class：类，是对象的蓝图，定义了对象的属性和方法。
  • Object：对象，类的实例，具有类定义的属性和方法。
  • Parameter：参数，函数或方法定义时声明的变量，用于接收调用时传递的值。
  • Argument：实参，调用函数或方法时传递给参数的具体值。
  • Return：返回，函数执行完毕后向调用者返回一个值。
  • Escape：转义。
  • Slash/Backslash：斜杠/反斜杠。
  • tilde：波浪号~

数据结构与算法

  • Array：数组，一种存储相同类型元素的有序集合。
  • List：列表，类似于数组，但通常更灵活，可存储不同类型的元素，如 Python 中的列表。
  • Stack：栈，遵循后进先出（LIFO）原则的数据结构。
  • Queue：队列，遵循先进先出（FIFO）原则的数据结构。
  • Tree：树，一种分层的数据结构，有一个根节点和多个子节点。
  • Graph：图，由节点和边组成的数据结构，用于表示对象之间的关系。
  • Sorting Algorithm：排序算法，如冒泡排序（Bubble Sort）、快速排序（Quick Sort）等，用于对数据进行排序。
  • Search Algorithm：搜索算法，如线性搜索（Linear Search）、二分搜索（Binary Search）等，用于在数据集合中查找特定元素。

编程工具与环境

  • **IDE (Integrated Development Environment)**：集成开发环境，提供编写、调试、编译等多种功能的软件开发环境，如 Visual Studio Code、PyCharm 等。
  • Compiler：编译器，将高级语言编写的源代码转换为机器语言的程序。
  • Interpreter：解释器，逐行解释并执行源代码的程序，如 Python 解释器。
  • Debugger：调试器，用于查找和修复程序中的错误，可设置断点、单步执行等。
  • Library：库，一组预先编写好的代码，可被其他程序引用以实现特定功能，如 Python 的 NumPy 库。
  • Framework：框架，提供了一套基础结构和规范，帮助开发者更高效地开发应用程序，如 Django 是 Python 的 Web 开发框架。

编程概念和操作

  • Loop：循环，让一段代码重复执行的结构，常见的有 for 循环、while 循环。
  • Conditional Statement：条件语句，根据条件的真假来决定执行不同的代码块，如 if...else、switch 语句。
  • Recursion：递归，函数调用自身的编程技巧，常用于解决具有递归性质的问题。
  • Exception：异常，程序运行过程中出现的错误或意外情况，如除零错误、文件未找到等。
  • Try - Catch：异常处理结构，用于捕获和处理程序中抛出的异常，避免程序崩溃。
  • Inheritance：继承，面向对象编程中，一个类可以继承另一个类的属性和方法。
  • Polymorphism：多态，不同对象对同一消息做出不同响应的能力，增强了代码的灵活性和可扩展性。
  • Encapsulation：封装，将数据和操作数据的方法绑定在一起，并隐藏对象的内部实现细节。

数据库相关

  • Database：数据库，用于有组织地存储、管理和检索数据的集合。
  • Table：表，数据库中存储数据的基本结构，由行和列组成。
  • Column：列，表中的垂直字段，每列存储特定类型的数据。
  • Row：行，表中的水平记录，包含了一组相关的数据项。
  • Primary Key：主键，表中用于唯一标识每一行记录的字段或字段组合。
  • Foreign Key：外键，用于建立表与表之间关系的字段，它引用了另一个表的主键。
  • **SQL (Structured Query Language)**：结构化查询语言，用于管理和操作数据库的标准语言。
  • Query：查询，使用 SQL 语句从数据库中检索数据。
  • Insert：插入，向数据库表中添加新记录的操作。
  • Update：更新，修改数据库表中现有记录的数据。
  • Delete：删除，从数据库表中移除记录。
  • Join：连接，将多个表中的数据根据一定条件组合在一起。

前端开发相关

  • **HTML (HyperText Markup Language)**：超文本标记语言，用于创建网页的结构。
  • **CSS (Cascading Style Sheets)**：层叠样式表，用于设置网页的外观和布局。
  • JavaScript：一种用于为网页添加交互性的脚本语言。
  • **DOM (Document Object Model)**：文档对象模型，将网页文档表示为树形结构，允许 JavaScript 动态操作网页内容。
  • **BOM (Browser Object Model)**：浏览器对象模型，提供了与浏览器窗口进行交互的对象和方法。
  • Responsive Design：响应式设计，使网页能够自适应不同设备的屏幕尺寸。
  • Animation：动画，通过 CSS 或 JavaScript 为网页元素添加动态效果。
  • Event：事件，如点击、鼠标移动、键盘输入等，可触发 JavaScript 代码执行。
  • Component：组件，将网页的一部分封装成可复用的模块，便于开发和维护。

后端开发相关

  • Server：服务器，为客户端提供服务的计算机或软件程序。
  • **API (Application Programming Interface)**：应用程序编程接口，允许不同的软件系统之间进行交互和数据共享。
  • RESTful API：遵循 REST 架构风格的 API，使用 HTTP 请求方法（如 GET、POST、PUT、DELETE）进行资源操作。
  • Middleware：中间件，在服务器和应用程序之间处理请求和响应的软件层，可用于身份验证、日志记录等。
  • Deployment：部署，将开发完成的应用程序发布到生产环境中运行。
  • Scalability：可扩展性，系统能够处理不断增长的用户请求和数据量的能力。
  • Load Balancing：负载均衡，将客户端请求均匀分配到多个服务器上，以提高系统的性能和可靠性。

版本控制相关

  • **Version Control System (VCS)**：版本控制系统，用于管理代码的版本历史，允许开发者跟踪代码的修改、协作开发等。
  • Git：一种分布式版本控制系统，广泛应用于软件开发中。
  • Repository：仓库，存储代码和版本历史的地方。
  • Branch：分支，在仓库中创建的独立开发线路，可同时进行不同功能的开发。
  • Merge：合并，将不同分支上的代码更改合并到一起。
  • Commit：提交，将本地的代码更改保存到版本控制系统中，并记录更改的描述信息。
  • Push：推送，将本地仓库的提交更新到远程仓库。
  • Pull：拉取，从远程仓库获取最新的代码更新并合并到本地仓库。
  • Conflict：冲突，在合并代码时，由于不同分支对同一部分代码进行了不同的修改而产生的冲突。
