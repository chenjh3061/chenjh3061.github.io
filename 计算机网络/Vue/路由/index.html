Vue路由

在Vue.js中，route和router是Vue Router库中的两个核心概念，它们分别代表不同的东西，主要用于处理页面的导航和路由管理。useRoute和useRouter是Vue Router中的组合式API，用于在Vue 3的组合式API（Composition API）中使用路由功能。 

route vs router 

route 

  • 定义: route对象表示当前激活的路由信息。它包含了关于当前路由的所有信息，如路径、参数、查询参数等。 

  • 属性: 

    • path: 当前路由的路径 

    • params: 动态路由匹配的参数 

    • query: URL中的查询参数 

    • name: 命名路由的名称 

    • fullPath: 完整的URL路径 

    • meta: 路由元信息 

    • 等等

 

router 

  • 定义: router对象是Vue Router的实例，包含了全局的导航方法和路由配置信息。通过它可以进行编程式导航和路由管理。 

  • 方法: 

    • push(location): 导航到一个新的URL 

    • replace(location): 替换当前的URL 

    • go(n): 在历史记录中前进或后退 

    • back(): 后退一步 

    • forward(): 前进一步 

    • 等等

     

  • 属性: 

    • options: 路由器的配置对象 

    • currentRoute: 当前激活的路由对象

 

useRoute vs useRouter 

useRoute 

  • 定义: useRoute是Vue Router的组合式API，用于在组合式API函数组件中访问当前的路由信息。它相当于访问this.$route，但是在组合式API中使用。 

  • 使用场景: 当你需要在Vue 3的组合式API中访问当前路由信息时使用。 

  • 示例: import { useRoute } from ‘vue-router’;  export default {   setup() {     const route = useRoute();     console.log(route.path);     return { route };   } };

  

useRouter 

  • 定义: useRouter是Vue Router的组合式API，用于在组合式API函数组件中访问路由实例。它相当于访问this.$router，但是在组合式API中使用。 

  • 使用场景: 当你需要在Vue 3的组合式API中进行编程式导航或访问路由实例时使用。 

  • 示例: import { useRouter } from ‘vue-router’;  export default {   setup() {     const router = useRouter();     const navigateToHome = () => {       router.push(‘/‘);     };     return { navigateToHome };   } };

  

详细解释 

  1. route对象: 

    • 包含当前激活的路由信息。 

    • 可用于获取当前路径、查询参数、动态路由参数等。 

    • 示例: const route = this.$route; console.log(route.path); // 输出当前路径 console.log(route.params); // 输出当前路由参数

     

  2. router对象: 

    • 是Vue Router的实例，用于管理应用的路由。 

    • 提供导航方法来编程式地导航到不同的路由。 

    • 示例: const router = this.$router; router.push(‘/home’); // 导航到/home路径

     

  3. useRoute组合式API: 

    • 用于在组合式API中访问当前路由信息。 

    • 示例: import { useRoute } from ‘vue-router’;  export default {   setup() {     const route = useRoute();     console.log(route.path); // 输出当前路径     return { route };   } };

      

  4. useRouter组合式API: 

    • 用于在组合式API中访问路由实例。 

    • 示例: import { useRouter } from ‘vue-router’;  export default {   setup() {     const router = useRouter();     const navigateToHome = () => {       router.push(‘/home’); // 导航到/home路径     };     return { navigateToHome };   } };

Vue Router(v4.x) 入门指南

Vue Router 是 Vue.js 的官方路由。它与 Vue.js 核心深度集成，让用 Vue.js 构建单页应用变得轻而易举。功能包括：

●嵌套路由映射

●动态路由选择

●模块化、基于组件的路由配置

●路由参数、查询、通配符

●展示由 Vue.js 的过渡系统提供的过渡效果

●细致的导航控制

●自动激活 CSS 类的链接

●HTML5 history 模式或 hash 模式

●可定制的滚动行为

●URL 的正确编码

 动态路由传参

很多时候，我们需要将给定匹配模式的路由映射到同一个组件。例如，我们可能有一个 User 组件，它应该对所有用户进行渲染，但用户 ID 不同。在 Vue Router 中，我们可以在路径中使用一个动态字段来实现，我们称之为路径参数

现在像 /user/123 和 /user/hello 这样的 URL 都会映射到同一个路由。

路径参数 用冒号 : 表示。当一个路由被匹配时，它的 params 的值将在每个组件中以 this.$route.params 的形式暴露出来。

一些应用程序的 UI 由多层嵌套的组件组成。在这种情况下，URL 的片段通常对应于特定的嵌套组件结构。

第六章 命名视图

命名视图可以在同一级（同一个组件）中展示更多的路由视图，而不是嵌套显示。 命名视图可以让一个组件中具有多个路由渲染出口，这对于一些特定的布局组件非常有用。 命名视图的概念非常类似于“具名插槽”，并且视图的默认名称也是 default。

一个视图使用一个组件渲染，因此对于同个路由，多个视图就需要多个组件。确保正确使用 components 配置 (带上 s)
