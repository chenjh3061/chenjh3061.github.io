JS解构赋值

发表于 2024-08-30 分类于 前端 阅读次数： 本文字数： 1k 阅读时长 ≈ 1 分钟
tip之解构赋值

ES6新引入的语法，可以从数组和对象中提取值来赋值，这种方法成为解构（destructuring。 

如：
![[Pasted image 20240830164623.png]]

本质上，这种写法属于“模式匹配”，只要等号两边的模式相同，左边的变量就会被赋予对应的值。 

如：let [foo, [[bar], baz]] = [1, [[2], 3]]; 

注意这里右边必须要是可遍历的结构，否则报错。 
解构不成功就会出现undefined。不完全解构(比如变量多于值)，也可以成功，只是只有部分赋到值。 

数组解构的妙用：
![[Pasted image 20240830164648.png]]

对象解构赋值的例子：

![[Pasted image 20240830164707.png]]

嵌套的对象同理。 

对象的解构与数组有一个重要的不同。数组的元素是按次序排列的，变量的取值由它的位置决定；而对象的属性没有次序，变量必须与属性同名，才能取到正确的值。如果 没有对应同名属性，就会变成undefined。 

解构赋值允许默认值， 

  1. let [foo = true] = []; 

  2. foo // true 

  3. let [x, y = ‘b’] = [‘a’]; // x=’a’, y=’b’ 

  4. let [x, y = ‘b’] = [‘a’, undefined]; // x=’a’, y=’b’

 

注意，ES6 内部使用严格相等运算符（===），判断一个位置是否有值。所以，只有当一个数组成员严格等于undefined，默认值才会生效。 

  1. let [x = 1] = [undefined]; 

  2. x // 1 

  3. let [x = 1] = [null]; 

  4. x // null

 

注意ES6会把句子开头的{理解成代码块，最好加括号：({x} = {x: 1});

![[Pasted image 20240830164724.png]]

ES6 的规则是，只要有可能导致解构的歧义，就不得使用圆括号。 

但是，这条规则实际上不那么容易辨别，处理起来相当麻烦。因此，建议只要有可能，就不要在模式中放置圆括号。
