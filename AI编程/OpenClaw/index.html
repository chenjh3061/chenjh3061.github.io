📖 一、项目介绍

1.1 什么是 OpenClaw？

OpenClaw（前身名为 Clawdbot、过渡名 Moltbot）是一款开源、本地优先的 AI 智能体（AI Agent）框架，由奥地利开发者 Peter Steinberger（PSPDFKit 创始人）创建，核心使用 TypeScript 编写。

2026年1月首次爆火，短短约60天内 GitHub Star 数突破 25 万（3月3日），随后于3月22日以 310,000+ Stars 全面超越拥有十余年积累的 React，成为 GitHub 历史上增长最快、Star 数最多的开源软件项目。截至2026年4月，Star 数持续攀升。项目图标为红色龙虾（🦞），口号是 **”The AI that actually does things”**（真正能干活的 AI），因此被网友戏称为”龙虾”或”养龙虾”。

核心理念： OpenClaw 的核心突破在于将大语言模型的”智能大脑”与本地执行环境的”行动手脚”无缝结合，实现了从”被动对话”到主动执行的范式转变。您可以将其理解为一个由大语言模型驱动的、24小时在线的”数字员工”——不只是帮你”出主意”，而是直接替你”干活”。

1.2 核心定位

OpenClaw是一个本地优先的开源AI Agent平台，支持在用户自己的计算机或服务器上部署。它能够理解自然语言指令，并在授权范围内执行系统操作、自动化流程和任务调度。与以云端服务为核心的AI工具不同，OpenClaw更关注长期使用的稳定性、数据控制权以及与真实工作流程的融合。

1.4 项目背景

OpenClaw 在2026年初的 AI 技术浪潮中以惊人的速度席卷全球开发者社区。创始人 Peter Steinberger 是业内知名的 PSPDFKit 开源库作者，其工程基因让 OpenClaw 从一开始就具备极高的可扩展性和工程质量。

该项目实现了从”对话式建议”到”自动化执行”的跨越，能够接管键鼠、直接调用系统 API，并完成读取文件、浏览器自动化、调用工具、调试代码等多流程复杂任务。其技术架构以 Gateway（网关层）→ Channel（消息路由层）→ LLM（模型推理层）→ Skill（技能执行层） 四层设计为核心，每层职责清晰、可灵活替换。

🔧 五、Skill开发指南

5.1 什么是Skill？

Skill（技能）是OpenClaw的核心扩展机制，让AI Agent能够执行特定的任务。可以将Skill理解为OpenClaw的”技能包”，它定义了AI如何识别用户意图并执行相应操作。
本质是封装特定任务知识、流程与执行逻辑的标准化、可复用能力

三个类比帮助理解：

  • 智能手机App： 每个Skill就像一个专门的应用，为OpenClaw添加新能力
  • 编程函数库： Skill是预定义的功能模块，AI可以调用执行特定任务
  • 技能工具箱： 每个Skill是一个专业工具，让AI具备特定领域的操作能力

5.2 Skill体系架构

Skills加载优先级

Skills从三个位置加载，名称冲突时按优先级覆盖：

位置 说明 优先级 适用场景
<workspace>/skills/ 每个Agent私有 ⭐⭐⭐ 最高 项目专属Skill
~/.openclaw/skills/ 所有Agent共享 ⭐⭐ 中等 通用Skill
内置Skills OpenClaw默认 ⭐ 最低 基础功能

Skill触发机制

  1. 用户输入解析： AI分析用户指令
  2. Skill匹配： 根据SKILL.md中的描述匹配最合适的Skill
  3. 权限验证： 检查Skill是否具备所需权限
  4. 执行任务： 调用Skill的代码逻辑
  5. 结果反馈： 将执行结果返回给用户
