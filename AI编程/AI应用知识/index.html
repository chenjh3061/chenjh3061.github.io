SKILL

Skill（技能）是指封装特定任务知识、流程与执行逻辑的标准化、可复用能力模块，是 AI Agent 完成复杂任务的核心执行单元。这一概念由Anthropic公司于2025 年正式提出并标准化，现已成为主流 AI 产品的核心能力范式。

提示词工程

提示词工程应用于开发和优化提示词（Prompt），帮助用户有效地将语言模型用于各种应用场景和研究领域。能够让AI更准确理解需求、高效完成任务。
核心价值
1.提升模型准确性
·减少模糊性和歧义
·明确任务边界和期望输出格式
·提高回答的精确度和可靠性
2.控制输出质量
·指定输出风格 (正式/非正式、技术/通俗)
·控制输出长度和详细程度
·确保内容符合特定标准
3.解锁高级能力
·引导模型进行复杂推理
·实现多步骤任务执行
·激发模型的创造性思维

提示词可以包含以下任意要素：
指令：想要模型执行的特定任务或指令。

输入数据：用户输入的内容或问题。
输出指示：指定输出的类型或格式。

尽可能用精准指令、角色扮演、结构化输出。

名词解释

Token

AI理解语言的基础，通过切分语句形成。

对于中文来说，1个汉字大约等于 1.5-2 个 Token；对于英文，1个单词大约等于 1-2 个 Token。比如”你好世界”大约是6-8 个 Token，”Hello World”大约是 2-3 个 Token。不同的模型使用不同的分词方式，Token 数量会有差异。

Stop Sequences：stop sequence 是一个字符串，可以阻止模型生成 token，指定 stop sequences 是控制大模型响应长度和结构的另一种方法。例如，您可以通过添加 “11” 作为 stop sequence 来告诉模型生成不超过 10 个项的列表。

Frequency Penalty：frequency penalty 是对下一个生成的 token 进行惩罚，这个惩罚和 token 在响应和提示中已出现的次数成比例， frequency penalty 越高，某个词再次出现的可能性就越小，这个设置通过给 重复数量多的 Token 设置更高的惩罚来减少响应中单词的重复。

Presence Penalty：presence penalty 也是对重复的 token 施加惩罚，但与 frequency penalty 不同的是，惩罚对于所有重复 token 都是相同的。出现两次的 token 和出现 10 次的 token 会受到相同的惩罚。 此设置可防止模型在响应中过于频繁地生成重复的词。 如果您希望模型生成多样化或创造性的文本，您可以设置更高的 presence penalty，如果您希望模型生成更专注的内容，您可以设置更低的 presence penalty。

与 temperature 和 top_p 一样，一般建议是改变 frequency penalty 和 presence penalty 其中一个参数就行，不要同时调整两个。

Content Window：上下文窗口，大模型一次能处理的最大Token量，包括提示词、对话历史、当前查询和输出内容，是模型的短期记忆容量。开头结尾大模型比较注意，可以关键指令开头、重要数据放结尾。

MCP

model context protocol模型上下文协议，AI大模型的标准化工具箱，是AI和外部工具的中间层。本质上是一个Node.js程序。

温度 (Temperature)

温度参数控制模型生成文本的随机性和创造性程度（默认值为0.7）。具体表现为：

  • 低温度值(0-0.3)：

    • 输出更确定、更专注
    • 适合代码生成、数据分析等需要准确性的场景
    • 倾向于选择最可能的词汇输出

  • 中等温度值(0.4-0.7)：

    • 平衡了创造性和连贯性
    • 适合日常对话、一般性写作
    • 推荐用于聊天机器人对话(0.5左右)

  • 高温度值(0.8-1.0)：

    • 产生更具创造性和多样性的输出
    • 适合创意写作、头脑风暴等场景
    • 但可能降低文本的连贯性

Top P (核采样)

默认值为 1，值越小，AI 生成的内容越单调，也越容易理解；值越大，AI 回复的词汇范围越大，越多样化。

核采样通过控制词汇选择的概率阈值来影响输出：

  • 较小值(0.1-0.3)：

    • 仅考虑最高概率的词汇
    • 输出更保守、更可控
    • 适合代码注释、技术文档等场景

  • 中等值(0.4-0.6)：

    • 平衡词汇多样性和准确性
    • 适合一般对话和写作任务

  • 较大值(0.7-1.0)：

    • 考虑更广泛的词汇选择
    • 产生更丰富多样的内容
    • 适合创意写作等需要多样化表达的场景

SOP：（Standard Operating Procedure，标准作业程序） 是指为确保 AI 项目、模型开发、部署或运维过程规范化、可重复、可监控而制定的标准化流程文档。它用于明确每个环节的操作步骤、责任分工、质量标准和风险控制，以提升效率、降低失误，并保障 AI 系统的可靠性、安全性和合规性。

双流水线算法

将任务拆分成两个独立的流水线，通过并行处理、资源隔离与动态协调，实现计算效率或吞吐量的显著提升。比如deepseek用这个来同时处理输入和输出。
